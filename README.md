АВТОМАТИЗАЦИЯ ТЕСТИРОВАНИЯ: SWAG LABS

1. ОПИСАНИЕ ПРОЕКТА
------------------

ЭТОТ ПРОЕКТ ПРЕДСТАВЛЯЕТ СОБОЙ ФРЕЙМВОРК АВТОМАТИЗИРОВАННОГО E2E ТЕСТИРОВАНИЯ ДЛЯ ВЕБ-ПРИЛОЖЕНИЯ SWAG LABS (ИНТЕРНЕТ-МАГАЗИН).

ФРЕЙМВОРК РЕАЛИЗОВАН С ИСПОЛЬЗОВАНИЕМ ПАТТЕРНА PAGE OBJECT MODEL (POM) И ПРОВЕРЯЕТ КЛЮЧЕВЫЕ БИЗНЕС-ПРОЦЕССЫ, ВКЛЮЧАЯ АВТОРИЗАЦИЮ, КАТАЛОГ, КОРЗИНУ И ПОЛНЫЙ ПРОЦЕСС ОФОРМЛЕНИЯ ЗАКАЗА (CHECKOUT).

2. ТЕХНОЛОГИИ
-------------

ЯЗЫК: JAVA
СБОРЩИК: APACHE MAVEN
ФРЕЙМВОРК ТЕСТИРОВАНИЯ: JUNIT 5
АВТОМАТИЗАЦИЯ: SELENIUM WEBDRIVER
УПРАВЛЕНИЕ ДРАЙВЕРОМ: WEBDRIVERMANAGER
БРАУЗЕР: THORIUM (НАСТРАИВАЕМЫЙ)

3. СТРУКТУРА ПРОЕКТА
--------------------

PROJ/
  |-- src/test/java/
      |-- base/           (BASETEST.JAVA: НАСТРОЙКА И ЗАКРЫТИЕ WEBDRIVER)
      |-- pages/          (PAGE OBJECTS: LOGINPAGE, CARTPAGE, CHECKOUTPAGE И ДР.)
      |-- tests/          (ТЕСТОВЫЕ КЛАССЫ: LOGINTESTS, CHECKOUTTESTS И ДР.)
      |-- utils/          (CONFIGREADER.JAVA: ЧТЕНИЕ НАСТРОЕК)
  |-- src/test/resources/ (CONFIG.PROPERTIES: КОНФИГУРАЦИОННЫЕ ДАННЫЕ)
  |-- pom.xml             (ЗАВИСИМОСТИ MAVEN)

4. НАСТРОЙКА И ЗАПУСК
---------------------

ПРЕДВАРИТЕЛЬНЫЕ УСЛОВИЯ:
  - УСТАНОВЛЕНЫ JDK 21 И MAVEN.
  - УСТАНОВЛЕН БРАУЗЕР THORIUM ИЛИ CHROME.

КОНФИГУРАЦИЯ:
  - ПРОВЕРЬТЕ И ПРИ НЕОБХОДИМОСТИ ИЗМЕНИТЕ ПУТЬ К БРАУЗЕРУ В ФАЙЛЕ:
    src/test/resources/config.properties
  - КЛЮЧ: browser.path

ЗАПУСК ВСЕХ ТЕСТОВ (КОНСОЛЬ):
  В КОРНЕВОЙ ДИРЕКТОРИИ ПРОЕКТА ВЫПОЛНИТЕ:
  mvn clean test

ЗАПУСК ЧЕРЕЗ INTELLIJ IDEA:
  - ЩЕЛКНИТЕ ПРАВОЙ КНОПКОЙ МЫШИ ПО ПАПКЕ 'src/test/java' И ВЫБЕРИТЕ 'RUN ALL TESTS'.

5. ТЕСТОВОЕ ПОКРЫТИЕ
--------------------

5.1. АВТОРИЗАЦИЯ (LOGINTEST.JAVA)
  - ПРОВЕРКА УСПЕШНОГО ВХОДА.
  - ПРОВЕРКА НЕВЕРНЫХ УЧЕТНЫХ ДАННЫХ.
  - ПРОВЕРКА ЗАБЛОКИРОВАННОГО ПОЛЬЗОВАТЕЛЯ.
  - ПРОВЕРКА ПУСТЫХ ПОЛЕЙ.

5.2. КАТАЛОГ ТОВАРОВ (INVENTORYTESTS.JAVA)
  - ДОБАВЛЕНИЕ/УДАЛЕНИЕ ТОВАРА В КОРЗИНУ.
  - ПРОВЕРКА ФУНКЦИИ СОРТИРОВКИ.

5.3. КОРЗИНА (CARTTESTS.JAVA)
  - ПЕРЕХОД НА СТРАНИЦУ КОРЗИНЫ.
  - УДАЛЕНИЕ ТОВАРА СО СТРАНИЦЫ КОРЗИНЫ.
  - ПЕРЕХОД К ОФОРМЛЕНИЮ ЗАКАЗА (CHECKOUT).

5.4. ОФОРМЛЕНИЕ ЗАКАЗА (CHECKOUTTESTS.JAVA)
  - ПОЛНЫЙ СКВОЗНОЙ ТЕСТ УСПЕШНОГО ЗАКАЗА.
  - ПРОВЕРКА ОШИБКИ ПРИ ПРОПУСКЕ ОБЯЗАТЕЛЬНЫХ ПОЛЕЙ.
  - ПРОВЕРКА КОРРЕКТНОСТИ ИТОГОВОЙ СУММЫ (PRICETESTS.JAVA).
